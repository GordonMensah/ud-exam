fix: body-text between repeated quoted words captured as scripture quote

Root cause: closing quote of first "muddy" acted as an opener for the
next _QUOTE_RE match, capturing all body text between the two uses of
the word "muddy" as a 180-char "scripture quote" for Proverbs 17:13.

Fix 1 - _NUMBERED_POINT_RE: add lookbehind for digit+space so labels
like 'f)' following a scripture ref ("Proverbs 17:13 f)") are correctly
detected as point boundaries. Previously 'f)' was missed because the
regex only looked for '. ' or ': ', not digit+space.

Fix 2 - Quote pairing filter: reject any captured quote containing an
internal sentence boundary. Real scripture quotes are single clauses;
body-text fragments between two quoted-word uses always have sentence
breaks. This ensures only genuine verse text is paired with refs.
