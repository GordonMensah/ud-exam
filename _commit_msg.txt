Fix _same_point_block off-by-one: use <= hi not < hi

Root cause: when a teaching point is extracted at pos P (= m.start()),
P is itself a boundary in all_point_boundaries. The old check
  lo < bp < hi  (strictly less than hi)
never triggered when bp == hi == P, so a ref from the PREVIOUS
point block was wrongly considered "in the same block" as the
teaching that STARTS the new block.

Fix: change to  lo < bp <= hi  so the boundary at the teaching's
own start position correctly separates it from earlier refs.

Fixes:
  - Ch10: "If you intend to set up a church..." (point g, no
    scripture) was incorrectly attributed to Ezekiel 34:18 (point f)
  - Ch11: "A murmurer must be driven away from the church" (point 4,
    Philippians 2:14) was incorrectly attributed to 1 Timothy 3:8
    (point 3)
